## 🎯 Общие требования к админке

### Доступ и авторизация
* **Единственный администратор:** Владелец проекта. Роли и уровни прав доступа не требуются.
* **Защита:** Авторизация по `telegram_id`. Список разрешенных ID зафиксирован в конфигурационном файле `settings.ADMIN_TELEGRAM_IDS`.
* **Интерфейс:** Выделенная веб-страница по пути `/admin` с обязательным входом через виджет Telegram.

### Интерфейсные паттерны
* Все списки сущностей должны поддерживать глобальный поиск, сортировку по ключевым колонкам и постраничную пагинацию.
* Для выбора связанных сущностей использовать выпадающие списки с автокомплитом (динамическая фильтрация при вводе).
* **Обязательные подтверждения (Модальные окна):**
  * Перед удалением любой сущности (окно "Вы уверены?").
  * Перед изменением поля `type` у предмета (предупреждение о потенциальном сбое логики и миграции).
  * Перед принудительным закрытием активного игрового сезона.

### Безопасность и целостность данных
* **Мягкое удаление (Soft Delete):** Прямое физическое удаление строк из таблиц (Hard Delete) строго запрещено. Используется флаг/таймстемп `deleted_at`.
* **Финансовые блоки:** Логи транзакций и истории покупок защищены от удаления на уровне СУБД (ограничения типа `RESTRICT`) и никогда не стираются.
* **Слой бизнес-логики:** Любые модификации данных должны проходить исключительно через Domain Use Cases приложения. Прямое выполнение SQL-запросов из контроллеров админки не допускается.

### Редактирование vs Пересоздание
* В приоритете обновление существующих записей через метод `PATCH`.
* При редактировании параметров предмета (название, описание, эффекты) текущий прогресс/инвентарь игроков сохраняется, обновляются только справочные данные.
* Запрещено изменение поля `type` у созданного предмета без сопутствующей миграции базы данных.

## 📦 Справочники и сущности

### 1. Артефакты (Artifacts)
Справочник пассивных модификаторов корабля. Один артефакт может содержать массив комплексных бонусов.

* **Поля:**
  * **Название:** String (Required) — Обязательное текстовое поле.
  * **Описание:** Text (Required) — Подробное описание эффекта.
  * **Картинка:** URL (Required) — Ссылка на графический ресурс.
  * **Редкость:** Enum (Required) — `common`, `uncommon`, `rare`, `epic`, `legendary`.
  * **Эффекты:** JSON (Required) — Массив структур (минимум 1). 
    * *Допустимые ключи:* `bonus_speed`, `bonus_defense`, `bonus_capacity`, `bonus_luck`, `bonus_fuel`, `bonus_repair`, `bonus_xp`, `bonus_fragment`.
    * *Значения:* Float (например, `0.15` означает `+15%`). Всего доступно 8 типов игровых слотов.
  * **Можно продать:** Boolean — Флаг доступности для продажи (`is_tradable`).
  * **Цена продажи:** Integer / Float — Стоимость в валюте `sell_price` (в XGen).
* **Важное бизнес-правило:** Вероятность выпадения артефакта в этой сущности не настраивается. Веса распределения указываются на стороне Локаций/Зон или Лутбоксов.
* **Операции:** Создать / Изменить (`PATCH`) / Удалить (`soft`). Вывод общего списка с фильтрацией по редкости.

### 2. Зоны экспедиций (Zones)
Игровые локации отправки флота с индивидуальными таблицами распределения лута.

* **Поля:**
  * **Название / Описание:** String / Text — Обязательные поля описания локации.
  * **Картинка:** URL — Ссылка на изображение зоны.
  * **fuel_cost:** Float (Required) — Расход топлива (чая) на экспедицию.
  * **optimism_risk:** Float (Required) — Риск повреждения корабля в диапазоне `[0.0 - 100.0]`.
  * **duration_seconds:** Integer (Required) — Продолжительность экспедиции в секундах.
  * **Лут-таблица (Массив дропов):** Список возможных наград. 
    * *Тип дропа:* `xgen`, `fragments`, `item`. Если выбран `item` — выбор объекта из общего справочника предметов.
    * *Параметры записи:* `amount` (количество), `chance` (вероятность Float от `0.0` до `1.0`).
  * **is_active:** Boolean — Статус доступности зоны для игроков.
* **Операции:** Создать / Изменить (`PATCH`) / Удалить (`soft`).
* **Инструментарий админа:** Кнопка "Preview лута" — запуск симуляции 100 экспедиций для оценки баланса дропа.
* **Защита:** Удаление зоны блокируется, если у игроков есть незавершенные экспедиции в данной локации.

### 3. Путеводитель — Статьи (Articles)
Лор-контент и информационные материалы, открываемые игроками за игровую активность или ресурсы.

* **Поля:**
  * **Название / Содержание:** String / Текст в формате Markdown (Required).
  * **Принадлежность к главе:** ID (Required) — Связь с родительской Главой (выбор из списка существующих).
  * **Тип разблокировки (Строго одно из):**
    1. **За фрагменты:** `fragment_cost` (Int > 0)
    2. **За триггер:** `trigger_event_type` + `trigger_threshold`
    3. **За ключевой предмет:** `required_item_id` (выбор предметов с типом `key_item`)
* **Расширяемый список триггеров:** `expedition_completed`, `article_unlocked`, `daily_login`, `chapter_completed`, `purchase_made`, `stars_donated`, `afk_seconds`.
* **Операции:** Создать / Изменить / Удалить (`soft`).
* **Защита:** Запрещено удалять статью, если её разблокировал хотя бы один пользователь (выводится системное предупреждение).

### 4. Путеводитель — Главы (Chapters)
Контейнеры для логической группировки статей путеводителя.

* **Поля:**
  * **Название / Описание:** String / Text (Required).
  * **Тип главы (Строго одно из):**
    * Обычная (`is_secret = false`, `season_id = null`)
    * Секретная (`is_secret = true`) — доступ только по триггерам
    * Сезонная (`season_id != null`) — жесткая привязка к таймлайну сезона
  * **Награда за закрытие главы (Конструктор наград):** `reward_xgen`, `reward_fragments`, массив предметов `reward_items` (структура `{item_id, quantity}` из категорий артефактов, расходников, лутбоксов или квест-итемов).
  * **Список статей:** Массив связанных статей с интерфейсом Drag-and-Drop для оперативного изменения порядка их отображения.
* **Операции:** Создать / Изменить / Удалить (`soft`), добавление/удаление статей из состава главы.
* **Защита:** Удаление главы невозможно, если в ней есть статьи, открытые пользователями.

### 5. Путеводитель — Сезоны (Seasons)
Временные игровые ивенты, инкапсулирующие уникальный контент.

* **Поля:**
  * **Название / Описание:** String / Text (Required).
  * **start_date / end_date:** DateTime UTC — Даты начала и окончания.
  * **Награды сезона:** Конструктор наград (`reward_xgen`, `reward_fragments`, `reward_items`).
  * **is_active:** Boolean — Ручное управление видимостью сезона.
* **Операции:** Создать / Изменить / Удалить (`soft`).
* **Защита:** Удаление активного во времени сезона заблокировано.

### 6. Лутбоксы (LootBoxConfigs)
Конфигуратор контейнеров со случайными наградами. Поддерживает создание неограниченного числа кастомных типов.

* **Поля:**
  * **Название / Описание:** String / Text (Required).
  * **box_type:** String (Required) — Уникальный строковый идентификатор. Предустановленные системные типы: `WELCOME`, `DAILY_42`, `CHAPTER_REWARD`, `SHOP`. Разрешено создание кастомных (например, `EVENT_HALLOWEEN_2026`).
  * **Картинка:** URL.
  * **is_active:** Boolean.
  * **Содержимое (Массив дропа):** Элементы типов `xgen`, `fragments`, `item` со своими количествами и шансами (`chance` от `0.0` до `1.0`). Сумма весов/шансов не обязана равняться 1.0 (каждая строка рассчитывается независимо).
* **Операции:** Создать / Изменить / Удалить (`soft`).
* **Инструментарий админа:** Встроенный симулятор — кнопка "Открыть 1000 боксов" для верификации распределения наград.

### 7. Магазин (ShopItems)
Интерфейс управления внутриигровыми продажами товаров за валюту XGen.

* **Поля:**
  * **Предмет:** Ссылка на справочник предметов (`Item`, Required).
  * **price_xgen:** Integer — Цена в XGen (>= 0).
  * **Лимиты:** `daily_limit` (в день на игрока, 0 — без лимита), `stock_limit` (общий глобальный лимит на всех, 0 — без лимита).
  * **is_active:** Boolean — Отображение на витрине.
  * **Скидка (Опционально):** `discount_percent` (0-100%) и дата окончания действия скидки `discount_ends_at` (DateTime).
  * **Набор / Бандл (Опционально):** `bundle_items` как массив структур `{item_id, quantity}`. Выдается целиком при покупке лота.
* **Операции:** Создать / Изменить / Удалить (`soft`). Фильтрация списков по статусам.
* **Аналитика:** Отображение счетчика успешных покупок сущности на основе исторических данных из таблицы `purchase_history`.

### 8. Пакеты Stars (StarsPackages)
Таблица монетизации: покупка игровой валюты XGen за Telegram Stars.

* **Поля:**
  * **Название пакета:** String (для UI, например, "Starter Pack").
  * **stars_amount:** Integer (> 0) — Цена в Stars.
  * **xgen_reward:** Integer (> 0) — Объем начисляемой валюты XGen.
  * **is_active:** Boolean.
* **Операции:** Создать / Изменить / Удалить (`soft`). Вывод общего списка.

### 9. Общий справочник предметов (Items)
Центральный реестр всех игровых предметов. Любой объект игры обязан присутствовать здесь.

* **Поля:**
  * Название, Описание, Картинка (URL).
  * **type (ItemType):** Зафиксирован при создании, изменение запрещено. Доступные типы:
    * `consumable` (расходники) — Формат эффектов: `{"restore_tea": 50}` или `{"restore_optimism": 30}`
    * `artifact` (артефакты) — Формат эффектов: `{"bonus_speed": 0.15, "bonus_luck": 0.05}`
    * `material` (материалы для крафта)
    * `key_item` (сюжетные ключи)
    * `loot_box` (контейнеры как предметы)
  * Редкость, Эффекты (JSON), Цена продажи.
* **Операции:** Создать / Изменить / Удалить (`soft`).
* **Жесткие ограничения безопасности (Валидация при удалении):** Запрещено удалять предмет из справочника, если он находится в инвентаре игроков, задействован в таблице дропа активных Зон/Лутбоксов или выставлен на продажу в Магазине.

## 🔒 Бизнес-правила (Обязательные)

1. **Soft Delete везде.** Физическое удаление данных полностью запрещено для сущностей, имеющих связи с игровым прогрессом.
2. **Финансы священны.** Таблицы `transactions` и `purchase_history` защищены на уровне БД через `ON DELETE RESTRICT`.
3. **Валидация на уровне Domain.** Все проверки бизнес-логики (проверка лимитов, валидация при удалении активных сезонов) реализуются строго внутри Use Cases, а не в HTTP-роутерах.
4. **Никаких прямых SQL-запросов.** Взаимодействие с базой данных в коде админки осуществляется только через абстракции Repository interfaces.
5. **Подтверждения.** Любое деструктивное действие (удаление, критическая смена параметров) требует вызова подтверждающего модального окна.
6. **Preview и симуляторы.** Для сущностей с недетерминированным результатом (Зоны, Лутбоксы) обязательно наличие инструментов симуляции в интерфейсе.

## 📅 Сезонная логика

* Сезоны переходят в статус активных автоматически на бэкенде, когда системное время `now >= start_date`.
* Сезоны автоматически завершаются, когда `now > end_date`.
* **Ручное управление:** Админодин может принудительно выставить флаг `is_active = false`, что мгновенно скрывает или деактивирует сезон независимо от установленного временного промежутка.
* **Отображение контента после закрытия:** Сезонные главы остаются доступными для просмотра и чтения тем игрокам, которые успели завершить их до наступления `end_date`. Для новых игроков или пользователей, не закрывших главу вовремя, данный контент полностью скрывается.

## 🎨 UI/UX Пожелания к интерфейсу

1. **Главный Дашборд:** Сводный экран с ключевыми бизнес-метриками (Общее число зарегистрированных пользователей, Daily Active Users (DAU), Суммарная выручка в Telegram Stars, Рейтинг популярности Зон экспедиций).
2. **Bulk-операции:** Инструменты массового редактирования (пакетное включение/выключение видимости предметов, скрытие из магазина).
3. **Импорт/Экспорт конфигураций:** Функционал сериализации настроек в JSON-файлы для быстрого бэкапа и развертывания конфигурации игрового мира.
4. **Журнал аудита (Audit Log):** Логирование действий администратора в системе (опционально, по запросу).
